【核心修复】连续点击发声持续问题
---------------------------------------------------------------------------

一、问题概述
当快速连续点击同一个钢琴键时，声音会持续播放无法停止。

二、相关知识背景

1. 竞态条件（Race Condition）
   - 定义：多个操作访问共享资源时，执行顺序不确定导致结果不可预测
   - 在JavaScript中：异步事件（mousedown/mouseup）的执行顺序不确定
   - 在多线程语言中：线程调度顺序不确定

2. 共享资源
   - activeSounds 对象：存储当前活动的声音对象
   - Web Audio API：独立于JavaScript主线程运行

3. 关键概念
   - 临界区：访问共享资源的代码段
   - 原子性：操作不可分割
   - 可见性：一个线程的修改对其他线程可见

三、问题根因分析

【问题1：声音对象注册延迟】
时序分析：
0ms:   mousedown → startNote() 开始执行
5ms:   创建音频节点（振荡器、增益节点、滤波器）
10ms:  osc.start() → 声音开始播放
15ms:  mouseup → stopNote() 被调用
20ms:  stopNote() 检查 activeSounds["C3"] → undefined（还没注册）
25ms:  startNote() 执行到 activeSounds["C3"] = {...}
结果：声音无法停止，因为 stopNote() 找不到声音对象

【问题2：定时器身份混淆】
时序分析：
0ms:   第一次点击 → 创建声音A，设置定时器T1（3秒后清理）
500ms: 第二次点击 → 创建声音B，设置定时器T2
3000ms: 定时器T1触发 → delete activeSounds["C3"]（此时是声音B）
结果：声音B被误删，后续 stopNote() 找不到声音

【问题3：Web Audio API独立运行】
- activeSounds 只是存储音频节点的引用
- delete activeSounds["C3"] 只是移除引用，不会停止音频
- 只有调用 osc.stop() 才能真正停止声音

四、解决方案

【方案1：提前注册声音对象】
在音频初始化前创建空对象并注册：
const soundData = { oscillators: [], gainNodes: [], soundId: uniqueId };
activeSounds[noteName] = soundData;  // 立即注册！
原理：保证 stopNote() 总能找到声音对象

【方案2：唯一ID验证】
为每个声音分配唯一ID，定时器只对匹配ID的声音生效：
if (activeSounds[noteName]?.soundId === currentSoundId) {
    delete activeSounds[noteName];
}
原理：防止旧定时器误删新声音

【方案3：立即静音】
停止旧声音时立即设置增益为0：
oldSound.gainNodes.forEach(gn => {
    gn.node.gain.setValueAtTime(0, ctx.currentTime);
});
原理：确保声音立即停止，零延迟

【方案4：取消旧定时器】
创建新声音前取消旧声音的清理定时器：
if (oldSound.cleanupTimer) {
    clearTimeout(oldSound.cleanupTimer);
}
原理：避免旧定时器干扰新声音

六、技术实现要点

1. 声音对象结构
{
    oscillators: [osc1, osc2, osc3],  // 振荡器数组（模拟泛音）
    gainNodes: [{ node: gn1, gain: 0.7 }, ...],  // 增益节点数组
    filters: [flt1, flt2, flt3],  // 滤波器数组
    startTime: ctx.currentTime,  // 创建时间
    soundId: "C3_1777136150032_a1d9lmdfy",  // 唯一ID
    cleanupTimer: null  // 清理定时器引用
}

2. startNote() 流程
- 生成唯一ID
- 检查并停止同名旧声音
- 提前注册空对象
- 创建音频节点并配置
- 连接音频图
- 应用ADSR包络
- 启动振荡器

3. stopNote() 流程
- 获取声音对象
- 计算释音时间
- 应用释音包络
- 调度振荡器停止
- 设置清理定时器
